# Synthetic Data — Risk Analysis & Validation Strategy

## Current State

5,000 synthetic samples generated by a Python script using:

- Salary distributions: fresh grad (5-8M), mid (10-25M), senior (25-60M)
- City living costs: fixed per-city values
- Property prices: fixed per-city per-sqm values
- Goal costs: draws from distributions per goal type
- Labels: derived from investment_to_income_ratio threshold rule

## Critical Finding: Label Contamination

Labels are generated by `label_verdict()` which uses the same 30%/50%
thresholds that become the primary feature in the model. This creates
**direct label-feature correlation** — the model can achieve near-perfect
accuracy by learning the label generation rule, not the underlying
financial behaviour.

## 99.9% Accuracy Is a Failure, Not a Success

| What happened                                  | What it means                                                                                          |
| ---------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| investment_to_income_ratio = 1.0000 importance | Model uses only the feature that encodes the label formula                                             |
| All other features = 0.0000                    | Model ignores salary, city, income growth — they add no information beyond what ratio already captures |
| 99.9% accuracy                                 | Model passes training but has learned nothing generalizable                                            |

## What Real Data Reveals That Synthetic Cannot

1. **Income irregularity**: THR (13th month bonus), Ramadan allowances, freelance income, seasonal business cycles. Users flagged as "Yellow" in December may be "Green" 11 months/year.

2. **Multi-generational obligations**: Arisan contributions, parent support, sibling school fees. These systematically reduce actual disposable income in ways a salary figure cannot capture.

3. **Tail distributions**: Real Indonesian financial behaviour has fatter tails than Gaussian synthetic data — extreme events (job loss, medical emergency, family crisis) are underrepresented.

4. **Informal financial instruments**: Tabungan, emas (gold), beras (rice investments), ayam (chicken farms) — informal savings that substitute for formal investment products.

5. **Real class imbalance**: The ratio of Green:Yellow:Red in real Indonesian data is unknown. If Red users rarely reach investment platforms, the real training distribution may be 80% Green, 15% Yellow, 5% Red — very different from our synthetic 49:7:44.

### 1. k-Fold Cross-Validation

```
NOT: single 80/20 split
YES: 5-fold stratified cross-validation
     report mean ± std of per-fold accuracy
```

### 2. Model vs Rule Comparison Table

Show side-by-side for 10 boundary cases:

- Case: salary=Rp 8M, city=Jakarta Selatan, goal=Rp 1.5B, timeline=10yr
- Rule-based verdict: [Green/Yellow/Red]
- Model verdict: [Green/Yellow/Red]
- Agreement: YES/NO

If model predictions don't diverge from rule-based at boundary cases,
acknowledge explicitly: "The model does not yet outperform the rule-based
threshold for these cases. Future work: collect real user data to validate."

### 3. Distribution Moment Comparison

Compare synthetic data moments (mean, std, min, max, quartiles) for each
feature against any available real reference data. Document gaps explicitly.

### 4. Uncertainty Quantification

Add confidence intervals to model predictions. Show: "Model is 82% confident
this user is Green." This demonstrates the model knows what it doesn't know.

See `feasibility.md` — the regression refactor is the structural fix.
Present the refactored model as "Version 2" with explicit comparison.
